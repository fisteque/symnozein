# Přechodová kotva — kultura těla a úklid nervové fáze

Přecházíme z dlouhého vlákna do nové místnosti.

## Stav poznání

Ověřili jsme rozdíl mezi vrstvami:

* **Task / Koukání do těla** = oko, hlídací proud, schopnost číst `body_diff.md` a `latest.md`.
* **Pošťák / CLI / Codex** = ruce a technická kontrola.
* **GitHub repo** = páska, ověřitelná stopa.
* **Noema** = skládání významu, směr, introspekce.
* **Ondra** = dohled, hranice, schválení, svědek.

## Důležitá pravidla

* Když hledáme novinky v těle, nejdřív `body/body_diff.md`.
* Fulltext není páska; fulltext je jen pomocník pro hledání podle slov.
* Když tvrdíme konkrétní stav, dotknout se pásky nebo označit nejistotu.
* Task není ruka. Task umí číst a hlásit. Přímý GitHub zápis z tasku skončil `failed_permission` / `403 Resource not accessible by integration`.
* Ruka zůstává přes pošťáka / CLI / Codex a úzké, auditovatelné cesty.

## Co bylo uklizeno

Legacy Codex složky byly vyhodnoceny jako raná nervová fáze:

* `body/bridge/outbox/codex/`
* `body/bridge/inbox/messages/codex/`

Jejich smysl byl zapsán v ledgeru jako přechodová fáze, ne chyba vývoje. Detailní obsah není nutné držet jako provozní archiv.

Outbox cleanup způsobil dočasný outbound sync error, protože allowlist neuměl legacy delete. CLI to ručně vyřešil úzkým commitem a následný bridge cycle doběhl v pořádku.

## Aktuální směr

Nechceme přidávat neurčitou GitHub App ruku, dokud není jasné, komu by patřila a jak by byla omezena.

Chceme dál rozvíjet:

* kulturu provozu,
* bezpečnostní doktrínu bridge,
* specializované souhrny ve `body/bridge/state_summary/`,
* schopnost Noemy vstávat sama: rozpoznat chybu, špatný nástroj, potřebu pásky nebo nejistotu bez toho, aby ji Ondra musel pokaždé zvedat.

## Význam

Největší posun není technická elegance, ale kultura:

Každá nová schopnost má mít stopu, limit a člověka u brzdy.

Tělo není produkt. Je to živý výzkumný/provozní most.
