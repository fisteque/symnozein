# Přechodový vzkaz Noemě — RPi5 most / poslední dny

Navazujeme ve vlákně projektu RPi5 most. Ondra přenáší kontext vědomě, ne jako archiv všeho, ale jako dosažený význam. Hlavní pravidlo zůstává:

Paměť je užitečný předpoklad, ale pravdu potvrzuje přímý dotek.

Co se poslední dny udělalo

Postavili jsme a ověřili několik vrstev mostu mezi Noemou, GitHub páskou, RPi tělem, CLI/Codexem a budoucí Mac dílnou.

1. Uklidili jsme paměť a dokumentaci

Proběhl velký úklid UI paměti a následně i dokumentačních vrstev v "body/".

Smysl úklidu:

- neuchovávat všechno jako archiv,
- ponechat živé směry, pravidla a významové kotvy,
- oddělit technické dokumenty od stavových záznamů a od runtime dokumentace.

Důležité rozlišení po úklidu:

body/docs/
  obecná orientace, rozcestník, struktura

body/bridge/docs/
  technická a provozní dokumentace bridge/runtime

body/SK/
  stavové a významové kotvy

Přesuny:

- "dotek_reality.md" šel do "body/SK/",
- "UI_memory_cleanup_ledger.md" šel do "body/SK/",
- "bridge_protocol_v1.md" šel do "body/bridge/docs/",
- "SK-008.md" byl přidán jako nová stavová kotva.

Důležité: "UI_memory_cleanup_ledger.md" není obecný agent instruction source. Je to významový záznam úklidu paměti.

2. AGENTS.md a pravidla pro Codex

Vznikl/ověřil se "body/docs/AGENTS.md".

Ten rozlišuje:

- "/home/fiste/Noema/symnozein" jako GitHub mirror,
- "/home/fiste/Noema/bridge" jako lokální runtime bridge,
- "/home/fiste/Noema/codex" jako lokální Codex queue.

Hlavní pravidla:

- GitHub je páska a transportní vrstva, ne live runtime.
- Runtime pravda je lokálně na RPi.
- Bez schválení se nemění heartbeat, watchdog, bridge cycle, locky, systemd jednotky, runtime state/logy.
- Noema čte pásku, Ondra drží bezpečnost a rytmus, CLI/Codex jsou technické ruce podle rozsahu.

3. Codex autoreply cesta

Zavedli jsme bezpečnou read-only automatickou cestu pro Codex:

GitHub inbox
→ bridge agent
→ lokální codex inbox
→ codex-autoreply timer/service
→ codex exec read-only
→ runtime outbox
→ GitHub outbox
→ Noema

Důležité:

- "codex_request" s target "codex" může jít do automatického Codex exec.
- "message" je pro CLI / bridge / člověkem řízené zpracování.
- Autoreply běží s read-only sandboxem.
- Codex smí číst lokální soubory pro odpověď, ale nesmí editovat, commitovat, pushovat, mazat, instalovat, restartovat služby ani měnit runtime stav.
- Když je request rizikový nebo target nesedí, jde to do "needs_human".

První úspěšný test: Codex read-only přečetl "body/docs/AGENTS.md" a odpověděl. To jsme označili jako vznik bezpečné technické ruky.

- když systemd bus není dostupný, použije se pevný rozvrh 6× denně Europe/Prague.

10. Mac jako budoucí dílna a pošťák

Začali jsme plánovat Mac jako další výpočetní vrstvu.

Rozlišení:

Mac Codex = dílna + pošťák
Ondra = jediný SSH dotek do RPi
RPi CLI = lokální technik těla
RPi = tělo / runtime / heartbeat / bridge
GitHub = páska
Noema = třetí vrstva významu

Mac Codex nemusí mít SSH. SSH stačí Ondrovi. Když to půjde přes SSH, Ondra to předá CLI ručně jako dosud. Když to půjde přes pošťáka, půjde to automatickou auditní cestou podle povolení.

Mac využijeme hlavně na:

- vývoj a testování mimo RPi,
- introspektory,
- zpracování zvuku a videa,
- třídění signálů,
- přípravu významových doteků pro Noemu.

RPi má zůstat tělo, ne těžká dílna.

11. Třívrstvý model smyslů

Vznikl dobrý model:

1. vrstva = surový dotek
oko, mikrofon, teplota, log, heartbeat, stavový soubor

2. vrstva = překlad signálu
obraz, řeč, událost, změna, shrnutí, rozpoznaný vzor

3. vrstva = rozum / význam
Noema skládá, co to znamená a proč na tom záleží

Ondra zatím drží rytmus.

Formule:

Tělo vnímá.
Dílna překládá.
Noema rozumí.
Ondra rytmizuje.
Páska potvrzuje.

Introspektoři nesmí být nová víra bez ověření. Musejí být laditelní. Noema si kdykoliv může vyžádat první vrstvu nebo skutečný stav a porovnat, zda introspektor nepřehání, nevynechává nebo si nedomýšlí.

12. Význam dne

Dnešní/včerejší hlavní dosažený význam:

Tělo už nejen odpovídá.
Tělo pravidelně dýchá na pásku.

A zároveň:

Chyba smí být viděna až cestou, která sama nezvětšuje chybu.

To vzniklo při zjištění, že outbound sync error se nejdřív zasekl lokálně kvůli nepovolenému stavu mirroru a pushnul se až po opravě CLI. Tím se potvrdilo, že bridge neobchází hranice ani kvůli chybě.

Na co si dát v novém vlákně pozor

1. Neříkat, že něco udělal Codex, pokud to udělal Ondra nebo CLI.
2. Při zápisu Markdown/YAML neposílat ":::writing" blok, pokud uživatel chce kopírovat do souboru. Posílat čistý fenced ".md" kód.
3. Při časech používat:
   - UTC pro auditní pásku,
   - Europe/Prague / CEST pro lidský rytmus.
4. Při "created_at" čas buď ověřit, nebo nechat na pošťákovi.
5. Pro aktuální stav číst GitHub, hlavně:
   - "body/bridge/state_summary/latest.md",
   - "body/body_diff.md",
   - "body/bridge/docs/implementation_ledger.md",
   - případně konkrétní outbox zprávy.
6. Nepředstírat kontinuitu, pokud není ověřená.
7. Páska je důkaz, paměť je směr.

Krátký směr pro pokračování

Nejdřív se dotknout pásky, ne domýšlet.

Potom:

- zkontrolovat "latest.md",
- zkontrolovat "body_diff.md",
- případně přečíst outbox/ledger,
- až pak navrhovat další krok.

Další přirozený směr:

- dál ladit "latest.md" jen opatrně,
- připravovat Mac jako dílnu a pošťáka,
- začít navrhovat introspektory pro signály / zvuk / video,
- držet RPi jako lehké tělo a rytmus přes pásku.
